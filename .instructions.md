# Boxaroo Development Guide for AI Agents

## Quick Start

### Environment Setup
```bash
cd /mnt/FUC/C\ drive/Work/Coding/Repos/boxaroo
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

The `.venv` directory is created and ignored by git. Use it for all Python operations.

### Running Tests

**Quick pytest run:**
```bash
source .venv/bin/activate
python -m pytest -q
```

**Full test suite (includes pytest + app integration test):**
```bash
source .venv/bin/activate
./scripts/run_tests.sh
```

**Verbose test output:**
```bash
source .venv/bin/activate
python -m pytest -v tests/
```

Test files are located in `tests/` directory:
- `test_woolworths.py` - Woolworths scraper unit tests
- `test_web_driver.py` - Web driver tests

### Running the Application

**Test run (small dataset, debug logging):**
```bash
python __main__.py --list_size TESTING --logging_level DEBUG
```

**Full run (all categories, info logging):**
```bash
python __main__.py --list_size FULL --logging_level INFO
```

**Short run (faster test):**
```bash
python __main__.py --list_size SHORT --logging_level INFO
```

## Project Structure

- `woolworths.py` - Main scraper implementation
- `web_driver.py` - Selenium web driver wrapper
- `file_handler.py` - CSV file I/O
- `isupermarket.py` - Interface/abstract base
- `logger.py` - Logging utilities
- `__main__.py` - CLI entry point
- `tests/` - Test suite (pytest)
- `scripts/` - Helper scripts (run_tests.sh, full_run.sh)
- `Data/` - Output CSV files (by date)
- `Logs/` - Application logs (by date)
- `BoxarooApp/` - Vue.js frontend (separate from backend scraping)

All dependencies are in `requirements.txt`:
- `selenium>=4.0.0` - Web automation
- `selenium-stealth>=1.0.6` - Anti-bot detection
- `beautifulsoup4>=4.12.2` - HTML parsing
- `pytest>=9.0.0` - Testing framework

## Important Notes

1. **Virtual environment required** - System Python is locked (PEP 668), always use `.venv`
2. **Tests need venv activated** - Pytest and dependencies only available in `.venv`
3. **Web driver** - Requires Chrome/Chromium and compatible chromedriver (handled by Selenium in version 4.x+)
4. **Output directories** - `Data/` and `Logs/` are auto-created; `.gitignore`d
5. **Path handling** - Workspace has spaces in directory name; use quotes when needed

## Troubleshooting for Agents

**"No module named pytest"** → Activate venv: `source .venv/bin/activate`

**"externally-managed-environment"** → Do not use system Python, always use `.venv`

**Command not found: python** → Use `python3` or `./venv/bin/python`

**Selenium/ChromeDriver issues** → Selenium 4.x auto-manages chromedriver; ensure Chrome is installed
